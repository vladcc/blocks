-- blocks 4.0 --
grep for nested data

Use: blocks [options] [files]

Prints proper nested blocks like so:
1. Match the name of the block, e.g. 'main'
2. Match the opening symbol, e.g. '{'
3. Match the corresponding closing symbol, e.g. '}'

Matching is done either as fixed strings or with the ECMAScript C++ std
regex library depending on the option provided before a <matcher> is
defined. The default for all <matcher>s is fixed string. Setting a <matcher>
to the empty string disables the <matcher>. I.e. setting the line comment to
"", or '', means blocks will consider line comments as ordinary input.

Note that the empty string should not touch the option string because there
is no way to distinguish between the empty string as an argument and the end
of the option string. I.e. use "-X ''" instead of "-X''".

Block start and block end are mandatory. If block name is not given, it is
set to block start.

blocks prints everything in-between the block name and the closing symbol.

For example if you have a file which looks like:
foo {
	bar {
		baz
	}
}

and you run 'blocks --block-name bar <my-file>', you'll get:
	bar {
		baz
	}
When the name is matched, the input isn't advanced and the search for
the block begins at the start of the name. This allows to match all blocks
in a file when the name is the same as the opening symbol. It also allows
to match LISP-like syntax where the block start comes before the name, e.g.:
blocks -n '(define' -s '(' -e ')'
Input is advanced when either block start or block end is matched.

Return codes:
help/version message:         0
No error, match was found:    0
No error, no match was found: 1
An error happened:            2

Unless stated otherwise, when multiples of the same option are given,
the last one applies.

Out of the plus options, f/r, i/A can appear together, e.g. +fi
Plus options a, o need to appear alone.

Options:
-n|--block-name <matcher>
Print only blocks starting with a name which matches <matcher>.
Default is '{'

-s|--block-start <matcher>
Set <matcher> as the open block symbol. Default is '{'

-e|--block-end <matcher>
Set <matcher> as the close block symbol. Default is '}'

-C|--comment <matcher>
Set <matcher> as the single line comment symbol.

-B|--block-comment-begin <matcher>
Set <matcher> as the start of block comment symbol.

-T|--block-comment-terminate <matcher>
Set <matcher> as the end of block comment symbol.

-m|--match <matcher>
Print only blocks in which at least one line matches <matcher>.
Does not ignore comments and strings.
If both match and don't match are given, logical and is applied even if
it is not specified on the command line. If a logical operation is
specified on the command line, both match and don't match must be give.
Logical operations:
+a - Logical and. Both matchers must match.
+o - Logical or. One or the other matcher must match.
+a|+o can appear anywhere on the command line.

-M|--dont-match <matcher>
Print only blocks in which no line matches <matcher>.
Does not ignore comments and strings.
If both match and don't match are given, logical and is applied even if
it is not specified on the command line. If a logical operation is
specified on the command line, both match and don't match must be give.
Logical operations:
+a - Logical and. Both matchers must match.
+o - Logical or. One or the other matcher must match.
+a|+o can appear anywhere on the command line.

-S|--mark-start <string>
When given, <string> will be printed before each block.

-E|--mark-end <string>
When given, <string> will be printed after each block.

-c|--block-count <num>
Print the first <num> number of blocks from the current file.

-k|--skip <num>
Don't print the first <num> number of blocks from the current file.

-l|--line-numbers
Print line numbers with output lines.

-N|--with-filename
Print file name with output lines.

-w|--files-with-match
Print only the names of files which contained at least one block.

-W|--files-without-match
Print only the names of files which did not contain a block.

-i|--case-insensitive
All matchers after this argument are case insensitive.
+i
Only the next matcher after this argument is case insensitive.

-A|--case-sensitive
All matchers after this argument are case sensitive.
+A
Only the next matcher after this argument is case sensitive.

-x|--no-defaults
Do not use defaults for matchers.

-I|--ignore-top
Print only the body of the block. I.e. don't print the block name,
block start, and block end. All lines between and including the block
name and block open, as well as the line containing the block end are
ignored.

-F|--fatal-error
Quit after the first error.

-V|--verbose-error
Print what was parsed along with the error message to stderr.

-D|--debug
Print debug info about matchers and quit.

-g|--lang <name>
Select language defaults. <name> is case insensitive and one of:
C, awk, json, xml, info
If used, all block matchers given on the command line but the block name
are ignored. info is the boost ptree info. xml takes care to place the
block name inside the tag automatically. To see the resulting matchers
for each language, use the debug info option. Ignores the no defaults
option.

-d|--directory <dir>
Read all files from <dir>. By default, all entries are read, not only files.
To filter, use the include/exclude options. Processed after the files given 
on the command line and before the file list option.
If both the include and the exclude options are provided, a file name must
match the include and not match the exclude in order to be selected.

-R|--recursive
Do a recursive file search. If the directory option is not provided, the
start directory defaults to '.'
Symlinks to directories are followed.

-u|--include-files-rx <regex>
If a file name found by the directory option matches <regex>, the file is
processed. <regex> is matched against the whole path. I.e. to make sure only
'.txt' files are filtered in, use '\.txt$'

-U|--exclude-files-rx <regex>
If a file name found by the directory option matches <regex>, the file is
not processed. <regex> is matched against the whole path. I.e. to make sure
only '.txt' files are filtered out, use '\.txt$'

-L|--file-list <file>
Read a list of input files from <file>. Processed after the files given
on the command line and after the directory option.

-z|--no-strings
Ignore strings when looking for matching patterns.

--string-rx <regex>
The regex used for identifying strings. Default is "([^\\"]|[\\].)*"
Used only if the no strings option is turned on.

-r|--regex-match
All matchers after this argument are regex.
+r
Only the next matcher after this argument is regex.

-f|--fixed-match
All matchers after this argument are fixed strings.
+f
Only the next matcher after this argument is fixed string.

-h|--help
Print this screen and quit.

-v|--version
Print version info and quit.
